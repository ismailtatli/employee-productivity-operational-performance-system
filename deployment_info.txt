Project Name:
Employee Productivity and Operational Performance Management System

Company / Scenario:
TatLee Factory

Project Type:
Role-based factory productivity and operational performance management web application.

GitHub Repository:
https://github.com/ismailtatli/employee-productivity-operational-performance-system

Backend Technology:
Node.js, Express.js, SQLite, JWT Authentication

Frontend Technology:
HTML, CSS, JavaScript

Backend Setup:
1. Open terminal.
2. Go to the backend folder:
   cd backend

3. Install dependencies:
   npm install

4. Start backend server:
   npm run dev

Backend URL:
http://localhost:5001

API Base URL:
http://localhost:5001/api

Frontend Setup:
1. Open another terminal.
2. Go to the frontend folder:
   cd frontend

3. Start local frontend server:
   python3 -m http.server 5501

Frontend URL:
http://localhost:5501

Role-Based Access Notes:
- System Admin has full system access.
- Factory Manager has full management and report access.
- Production Supervisor can access production-focused modules.
- Quality Control Specialist can access quality-related production data.
- HR Specialist can manage employees and departments.
- Department Viewer has dashboard-only read access.
- Reports are restricted to System Admin and Factory Manager.

Main Features:
- Secure login and token-based authentication.
- Role-based sidebar and dashboard.
- Employee management.
- Department management.
- Product management.
- Machine management.
- Production record management.
- Smart employee performance reports.
- Bonus eligibility and promotion candidate logic.
- Quality, continuity, on-time completion and performance scoring.
- Automated backend tests.
